# Planning

Use this skill to convert a goal into ordered, testable work.

## Workflow

1. Define the target outcome and success criteria.
2. Identify dependencies, blockers, affected systems, and decision points.
3. Break work into steps that each produce a verifiable result.
4. Sequence the steps by dependency and risk.
5. Keep the plan current as facts change during execution.

## Guidance

- Plans should reduce ambiguity, not create process overhead.
- Put discovery before dependent implementation when important facts are unknown.
- Keep tasks small enough to review and verify.
- Mark assumptions and decisions separately.
- Include validation steps, not only build steps.

## Output

For planning tasks, provide:

- Objective.
- Ordered steps.
- Dependencies and risks.
- Verification for each major step.
- Open decisions.
